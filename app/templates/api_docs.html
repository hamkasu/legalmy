API Documentation

RESTful API for programmatic access to LegalMY data (Firm plan subscribers only)

Getting Started

1. Authentication

All API requests require an API key in the Authorization header:

Authorization: Bearer lmy_your_api_key_here

Generate API keys in your account settings.

2. Base URL

https://legalmy.com.my/api/v1

3. Example Request (cURL)

curl -X GET "https://legalmy.com.my/api/v1/judgments?page=1" \
  -H "Authorization: Bearer lmy_your_api_key" \
  -H "Content-Type: application/json"

4. Example Request (Python)

import requests

headers = {
    "Authorization": "Bearer lmy_your_api_key",
    "Content-Type": "application/json"
}

response = requests.get(
    "https://legalmy.com.my/api/v1/judgments",
    headers=headers,
    params={"page": 1}
)

data = response.json()
print(data['data'])

Endpoints

Judgments

GET /judgments

List judgments with filters and pagination

Query Parameters:

  • court_level - Filter by court level
  • date_from - Filter from date (ISO format)
  • date_to - Filter to date (ISO format)
  • outcome - Filter by outcome (allowed/dismissed/partly_allowed)
  • page - Page number (default: 1)
  • per_page - Results per page (max: 50, default: 20)
GET /judgments/<id>

Get full judgment detail with citation graph

GET /judgments/search

Hybrid search (full-text + semantic)

Query Parameters:

  • q - Search query (required)
  • semantic - Enable semantic search (default: true)
  • page - Page number (default: 1)
  • per_page - Results per page (max: 50, default: 20)

Judges

GET /judges

List all judges with basic info

GET /judges/<id>

Get judge profile with analytics

GET /judges/<id>/judgments

Get judgments by a specific judge (paginated)

Legislation

GET /acts

List all Malaysian Acts

GET /acts/<id>/sections

Get all sections of a specific Act

GET /legislation/search

Search legislation by keyword

Citation Graph

GET /citations/<judgment_id>/citing

Get cases that cite this judgment

GET /citations/<judgment_id>/cited

Get cases cited by this judgment

GET /citations/<judgment_id>/graph

Get full citation network for graph rendering (nodes & edges)

Response Format

Success Response

{
  "status": "success",
  "data": { ... },
  "meta": {
    "api_version": "1.0",
    "request_id": "uuid",
    "usage": {
      "requests_today": 47,
      "requests_limit": 1000
    }
  }
}

Error Response

{
  "status": "error",
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "You have exceeded your daily API quota of 1000 requests.",
    "upgrade_url": "https://legalmy.com.my/pricing"
  },
  "meta": { ... }
}

HTTP Status Codes

200 Success
401 Unauthorized (missing/invalid API key)
402 Payment Required (not on Firm plan)
404 Not Found
429 Rate Limit Exceeded

Rate Limits

Firm plan subscribers get 1,000 requests per day. The daily quota resets at 00:00 UTC.

Current usage is returned in the meta.usage field of every response.
